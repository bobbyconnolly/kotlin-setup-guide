Step 1: Installing Java Development Kit (JDK)

While the official Kotlin documentation doesn't explicitly mention JDK installation, it's a required component for Kotlin development.

Download and Install OpenJDK 17 LTS:

Windows:

  1. Visit Adoptium
  2. Download OpenJDK 17 LTS
  3. Run the installer with default settings

macOS:

  1. Open Terminal
  2. Install Homebrew if not already installed
  3. Run: brew install --cask temurin17

Linux:

  1. Open Terminal
  2. Run: sudo apt install openjdk-17-jdk (Ubuntu/Debian)
  3. Or use your distribution's package manager

Verify Installation:

  1. Open Terminal/Command Prompt
  2. Run: java -version
  3. Expected output should show Java 17.x.x

  2. Accept the default installation location

  3. Right-click the app and select "Open" (important for first launch)

    ln -s /opt/intellij-idea-ce/bin/idea.sh ~/.local/bin/idea

Step 4: Verifying Kotlin Plugin

The Kotlin plugin should be pre-installed in recent versions of IntelliJ IDEA, but it's important to verify this.
  1. Open IntelliJ IDEA
  2. Access the Plugins section:
    Windows/Linux:
    1. File ‚Üí Settings
    2. Click Plugins in the left sidebar
    macOS:
    1. IntelliJ IDEA ‚Üí Settings
    2. Click Plugins in the left sidebar
  3. In the Plugins window:
    1. Click "Installed" tab
    2. Type "Kotlin" in the search bar
    3. Ensure the Kotlin plugin is listed and enabled

Step 5: Creating Your First Project

  1. From the welcome screen, click "New Project"
  2. In the New Project wizard:
    • Select "Kotlin" from the left sidebar
    • Choose "JVM | IDEA" as project target
    • Project name: KotlinLab01
    • Location: Choose your preferred directory
    • Language: Kotlin
    • Build system: Gradle
    • JDK: Select JDK 17 (installed earlier)
  3. Click "Create"
üìù Note: If JDK 17 isn't listed, click "Add JDK" and navigate to where you installed it:
  • Windows: Usually C:\Program Files\Java\jdk-17
  • macOS: Usually /Library/Java/JavaVirtualMachines/temurin-17.jdk
  • Linux: Usually /usr/lib/jvm/java-17-openjdk-amd64

Common Issues & Troubleshooting

‚ùå "Invalid JDK Configuration"

Solution:
  1. File ‚Üí Project Structure
  2. Under Project Settings ‚Üí Project:
    • Set SDK to JDK 17
    • Set Language Level to 17
  3. Click Apply and OK

‚ùå "Kotlin Not Configured"

Solution:
  1. Click "Configure Kotlin" in the notification
  2. Select "Add Kotlin to Build Script"
  3. Wait for Gradle sync to complete

‚ùå Environment Variable Issues

‚ö†Ô∏è IMPORTANT: If you see any errors related to environment variables:
  1. DO NOT modify system environment variables without consulting your instructor
  2. Take a screenshot of any error messages
  3. Note down what you were doing when the error occurred
  4. Contact your instructor with this information

‚ùå Gradle Sync Failed

Try these steps in order:
  1. Check internet connection
  2. File ‚Üí Invalidate Caches ‚Üí Invalidate and Restart
  3. Delete .gradle folder in project directory
  4. Re-sync project: View ‚Üí Tool Windows ‚Üí Gradle ‚Üí Refresh
